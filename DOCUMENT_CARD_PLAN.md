# FlashForge 本地资料制卡开发计划

## 目标

在现有 FlashForge 中增加本地资料制卡能力，支持从 Markdown 和 EPUB 中提炼值得记忆的知识，经过预览和编辑后复用现有 Anki 导入流程。

核心原则：

- 不把大文档整篇无结构地发送给模型。
- 课堂记录必须区分教师结论、学生回答、纠错过程和角色描写。
- 数学公式、标题、章节和来源信息必须保留。
- 大批量资料默认先预览，避免未经确认直接污染 Anki。
- 文档解析、知识提炼、卡片生成和 Anki 导入保持模块化。

## 第二阶段：候选知识点与批量 Markdown

- [ ] 增加 `KnowledgeCandidate` 中间模型，保存结论、解释、公式、误区、来源和置信度。
- [ ] 支持多文件和文件夹导入。

- [ ] 支持按核心知识、公式、错题、例题或课堂总结筛选。
- [ ] 跨文件去重并限制每章卡片数量。
- [ ] 在卡片标签或备注中保留来源文件和章节。

## 第三阶段：EPUB

- [ ] 安全解析 EPUB 容器、目录、章节和元数据。
- [ ] 保留标题、段落、列表、表格和 MathML 公式。
- [ ] 按目录层级切块，控制模型上下文和调用成本。
- [ ] 标记需要 OCR 或视觉模型处理的图片页。
- [ ] 支持章节选择、进度显示、失败重试和任务取消。

## 第四阶段：质量与规模化

- [ ] 卡片质量评分和低置信度提示。
- [ ] 数学公式与最终结论一致性检查。
- [ ] 导入历史、来源追踪和可恢复的批量任务。
- [ ] 更精细的 Anki 重复检测和跨牌组去重。
- [ ] 使用真实课堂记录、长 Markdown 和不同 EPUB 做端到端验收。

## 暂不纳入第一阶段

- 扫描版 PDF 和纯图片 EPUB。
- 全文向量索引或长期知识库。
- 多用户、云端同步和后台服务器。
- 未经预览直接批量导入整本书。
